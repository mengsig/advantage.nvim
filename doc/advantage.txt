1. INTRODUCTION                                       *advantage-introduction*

advantage.nvim is a coding-agent *harness* that runs its own agent loop inside
Neovim: it streams from the model, executes tools in your editor, feeds the
results back, and repeats until the task is done. It is not a wrapper around a
CLI.

- Runs on your subscription: Claude Code login (Pro/Max) or Codex / ChatGPT
  login. Environment API keys work as a fallback.
- Model-agnostic: Anthropic and OpenAI/Codex families out of the box.
- Editor-native tools executed inside Neovim, all edits reviewable.

==============================================================================
2. REQUIREMENTS                                       *advantage-requirements*

- Neovim >= 0.10
- `curl` on your PATH
- One of: a `claude` CLI login, a `codex` CLI login, or `$ANTHROPIC_API_KEY` /
  `$OPENAI_API_KEY`.
- Optional: `rg` (ripgrep) for fast grep/file listing; `wl-paste`/`xclip`/
  `pngpaste` for clipboard image paste.

==============================================================================
3. SETUP                                                     *advantage-setup*

Call `setup()` once (options are optional; defaults are sensible): >lua

    require("advantage").setup({
      default_model = "anthropic/claude-opus-4-8",
    })
<
With lazy.nvim: >lua

    { "mengsig/advantage.nvim", opts = {} }
<

4. COMMANDS                                               *advantage-commands*

                                                                   *:Advantage*
:Advantage              Toggle the chat panel.
:Advantage new          Start a new session.
:Advantage model        Pick the model.
:Advantage resume       Resume a saved session for this project.
:Advantage stop         Cancel the in-flight turn.
:Advantage usage        Token usage dashboard.
:Advantage compact [llm|heuristic]
                        Force conversation compaction. Defaults to
                        `context.compact_mode`; "llm" spends one call on
                        `context.summarizer_model` for a real summary,
                        "heuristic" stays free/offline.
:Advantage context      Show/manage per-repo memory (see |advantage-memory|).
:Advantage review       Review the agent's file changes (diff).
:Advantage yolo [on|off]
                        Toggle skip-all-permissions mode.
:Advantage effort       Tune reasoning effort / thinking.
:Advantage help         Keybinding & command cheatsheet.
:Advantage add          Send the current file / selection to the prompt
                        (with a range, e.g. `:'<,'>Advantage add`, sends
                        `@file:L10-20`).
:Advantage files        Pick a project file to add to the prompt.
:Advantage attach {path}
                        Attach a file (images become attachments, other files
                        `@mentions`). No path opens the file picker.
:Advantage ask {prompt} Send a prompt directly. With a visual range
                        (`:'<,'>Advantage ask why is this slow?`) the selected
                        lines are appended as a fenced block.

5. KEYMAPS                                                 *advantage-keymaps*

Defaults (configurable under `keymaps`):

  <leader>cc   toggle the chat panel
  <leader>cn   new session
  <leader>cm   pick model
  <leader>cr   resume session
  <leader>cs   add visual selection as an @file:L10-20 mention
  <leader>cf   send current file to the prompt
  <leader>cl   send @file:L{cursor line}
  <leader>cp   pick a project file to send
  <leader>cP   preview the exact context packet (system + tools + transcript)
  <leader>cu   usage dashboard
  <leader>cd   review file changes (diff)
  <leader>cy   toggle yolo mode
  <leader>ce   tune reasoning effort
  <leader>c?   keybinding cheatsheet

In the prompt: <CR> sends; `@` completes project files; `/` runs a slash
command; paste inserts clipboard images as attachment chips.

6. CONFIGURATION                                     *advantage-configuration*

Full defaults live in `lua/advantage/config.lua`. Key options: >lua

    require("advantage").setup({
      default_model = "anthropic/claude-opus-4-8",
      -- context_window scales compaction per model (adjust to your tier).
      models = { { ref = "anthropic/claude-opus-4-8", label = "opus 4.8",
                   context_window = 1000000 } },
      ui = { width = 0.42, input_height = 4, border = "rounded" },
      tools = {
        auto_approve = {},        -- e.g. { bash = true } (at your own risk)
        allow_outside_root = false,
        yolo = false,
        bash_timeout_ms = 120000,
      },
      context = {
        auto_compact = true,
        -- Auto-compact at min(compact_fraction × context_window, compact_at_tokens).
        compact_fraction = 0.75,   -- % of the model window that triggers compaction
        compact_at_tokens = 200000, -- absolute cost ceiling on that trigger
        keep_recent_messages = 16, -- also bounded by keep_recent_fraction (tokens)
        keep_recent_fraction = 0.4, -- recent window kept, as a % of the threshold
        auto_compact_mode = "heuristic", -- silent auto-compact: "heuristic" | "llm"
        compact_mode = "llm",    -- manual /compact: "llm" | "heuristic"
        summarizer_model = nil,    -- auto: Haiku for Claude, Luna for OpenAI/Codex
        summarizer_models = {
          anthropic = "anthropic/claude-haiku-4-5",
          openai = "openai/gpt-5.6-luna",
        },
      },
      subagents = { enabled = true, max_turns = 12, parallel = true },
      memory = { enabled = true, budget_tokens = 2000, skills_index_budget_tokens = 1200 },
      sessions = { autosave = true },
    })
<
Note: `models` and other list options are replaced wholesale when you supply
them (not merged element-wise with the defaults).

7. TOOLS & SECURITY                                          *advantage-tools*

Tools: `read_file`, `edit_file`, `write_file`, `multi_edit`, `bash`, `grep`,
`find_files`, `list_dir`, `diagnostics`, `sub_agent`, `web_search`, plus memory tools.

File tools are confined to the project root: absolute paths, `..` traversal,
and symlinks that resolve outside the repo are rejected (set
`tools.allow_outside_root = true` to opt out). Mutating tools (`edit_file`,
`write_file`, `multi_edit`, `bash`) require an approval card unless auto-approved
or `yolo` is on. Sub-agents are read-only.

After a mutating edit the newly-introduced LSP/linter diagnostics are appended to
that tool's result (errors only by default, capped, diffed against the pre-edit
state, silent on a clean edit) so the model self-corrects. A no-LSP repo pays no
per-edit cost and instead gets a once-per-filetype "install a language server"
notice — persisted as a line in the chat transcript (plus a WARN toast) so you
won't miss it if you stepped away. Configure via `tools.diagnostics` (`enabled`, `auto`, `severity`, `max`,
`wait_ms`, `attach_grace_ms`, `notify_missing`); the model also has an explicit
`diagnostics` tool.

`web_search` hits the Brave Search API (one GET request, no page-scraping) and
returns compact title/url/snippet results. It needs an API key — `$BRAVE_API_KEY`
or `tools.web_search.api_key` — and is entirely hidden from the schema without
one, so the model never calls a search tool that can't work. It's `safe` (no
approval prompt) and available to read-only sub-agents.

8. MEMORY & SKILLS                                          *advantage-memory*

advantage keeps per-repo memory under `<root>/.advantage/` in two tiers by cost:

  context.md            ALWAYS-LOADED crisp facts, one bullet each — the recurring
                        per-turn tax (rides the cached prefix), held to
                        `memory.budget_tokens` (default 2000). Keep it lean.
  skills/<name>/SKILL.md  DEPTH: a named procedure / deep-dive, index-only in
                        context until loaded on demand via use_skill. The index is
                        itself capped (`memory.skills_index_budget_tokens`) so a
                        large library never re-bloats the prefix.

Because the always-loaded tier is billed every turn, DEPTH belongs in skills, not
in facts: when a fact grows too verbose the `remember` tool steers the agent to
extract it into a skill (with a retrieval-rich description) and leave a crisp
pointer, and persistently nudges you to `/context curate`.

`:Advantage context` shows the memory; `context init` teaches the repo in one
pass; `context curate` compresses it (tighten facts, push depth into skills); `context verify` flags facts whose files
have moved/vanished; `context preview` (also `<leader>cP`) shows the exact
context packet that will be sent — system prompt, tool schemas and transcript —
with the cache boundary drawn and a per-section token breakdown, so you can see
what each part costs and confirm the memory block is frozen for cache reuse;
`context forget <text>` drops matching facts. Disable with
`memory = { enabled = false }`. AGENTS.md / CLAUDE.md are also ingested.

CUSTOM CONFIG DOCS                                    *advantage-config-docs*

Drop any Markdown file into `<root>/.advantage/` to add your own standing
instructions: every `.advantage/<name>.md` is injected verbatim into the system
prompt each turn (name-sorted for a cache-stable prefix, each capped by
`memory.config_budget_tokens`). No code change and no tool call is needed — e.g.
`.advantage/style.md` or `.advantage/review-rules.md`.

  NOTE: `context.md` CANNOT be used this way. It is the managed memory file
  (owned by the remember/curate machinery) and is excluded from config-doc
  ingestion. Put hand-authored instructions in any OTHER `.md` filename.
